Consider the QR decomposition performance for different matrix sizes. Below is a Python snippet that measures runtimes using NumPy (CPU) and CuPy (GPU). You can either click ‚ÄúRun Code‚Äù to reveal the CPU/GPU buttons, or select a single size below and click ‚ÄúRun Benchmark.‚Äù 

for size in [64, 128, 256, 512, 1024, 2048]:
    print("\nInput Matrix size: %d x %d" % (size, size))
    for xp in [np, cp]:
        A = xp.random.uniform(low=-1.0, high=1.0, size=(size,size)).astype(xp.float32)
        xp.linalg.qr(A)      # warm-up to exclude JIT
        start = perf_counter()
        xp.linalg.qr(A)
        cp.cuda.Device().synchronize()
        end = perf_counter()
        print(xp._name_, '%.6f' % (end - start))
    del A
